# セキュリティポリシー

歩み（Ayumi）は福祉事業所で **利用者の個人情報・要配慮個人情報** を扱います。
脆弱性の報告には特に感謝します。

## 対応バージョン

本プロジェクトは現在 0.x の開発初期段階です。修正は最新の `main` に対して提供します。

| バージョン | サポート |
| --- | --- |
| `main`（最新） | ✅ |
| それ以前 | ❌ |

## 脆弱性の報告

**脆弱性を公開の Issue に書かないでください。** 次のいずれかで非公開に連絡してください。

1. **GitHub の非公開報告**（推奨）: リポジトリの **Security → Report a vulnerability**
   （Private vulnerability reporting）から報告。
2. **メール**: silentmalachite@gmail.com 宛て。

報告には次を含めてください。

- 影響の概要と、想定される攻撃シナリオ
- 再現手順（最小の手順・必要なら PoC）
- 対象のバージョン／コミット

## 対応の流れ

- 受領後、できるだけ速やかに内容を確認し、状況を返信します。
- 修正方針が固まり次第、対応を進め、報告者に経過を共有します。
- 公開のタイミングは、利用事業所がアップデートできるよう報告者と調整します。

## 運用上の注意（導入者向け）

このアプリは **単一ホスト + LAN・オフライン** を前提に設計されています。安全に運用するため:

- SQLite ファイルをネットワーク共有（SMB/NFS）やクラウド同期フォルダ（OneDrive / iCloud /
  Dropbox）に置かないでください。同時アクセスでデータ破損の恐れがあります。
- LAN へ公開する場合でも、信頼できる事業所内ネットワークに限定してください。インターネットへ
  直接公開する用途は想定していません。
- 職員アカウントは `mix ayumi.create_user` で作成し、開発用シードの初期パスワードを本番で
  使い回さないでください。
- バックアップ（DB ファイル）は個人情報として適切に保護・管理してください。
