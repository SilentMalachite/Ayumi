How we work

進め方の原則

CLAUDE.md より。すべてのタスクはこの原則の上で進める。1つずつ・TDDで・mix review がグリーンになってから次へ。

壱

テスト先行（TDD）

失敗するテストを先に書く。context と changeset から着手する。

弐

追記専用（Append-only）

状態変更・訂正は新しい行として記録。既存行は上書きしない。履歴を失わない。

参

状態は導出する

「現在の状態」は保存せず、ログ最新行から純粋関数で導出。単体テスト容易に。

肆

薄い LiveView

ロジックは context へ。LiveView は assigns とイベントのみ。検証は changeset に集約。

伍

日本語 UI / 英語コード

UI 文言は日本語＋gettext。識別子・コメントは英語。

陸

最小差分・スコープ報告

差分は最小限。変更したファイル・関数とその理由を必ず報告する。

  • 参UI 文言は日本語＋gettext。識別子・コメントは英語。
  • 肆mix review とテストがグリーン。変更したファイル・関数と理由を報告。
